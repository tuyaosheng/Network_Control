# 主控端 HTTP 控制接口说明

主控端内置一个 HTTP 接口，供**其他程序**调用，实现「一键开网 / 一键禁网」。
接口随主控端 GUI 自动启动，无需额外进程，零额外依赖（Python 标准库实现）。

---

## 1. 基本信息

| 项目 | 默认值 | 说明 |
|------|--------|------|
| 监听地址 | `127.0.0.1` | 仅本机调用。需局域网其他机器调用时改为 `0.0.0.0` |
| 端口 | `8770` | 与 WebSocket（8765）分开 |
| 令牌 | 空（不校验） | 设置后写操作需携带令牌 |
| 协议 | HTTP，返回 JSON | 同时支持 `GET` 和 `POST` |

> 接口的动作与主控端界面上「**允许上网**」「**禁止上网**」两个按钮**完全等价**，
> 默认作用于**所有在线被控端**。

---

## 2. 接口列表

### 2.1 一键开网（允许全部上网）

```
POST /api/network/enable
GET  /api/network/enable      （也支持，便于浏览器/简单脚本调用）
```

等价于对所有被控端点击「允许上网」：停止过滤并恢复路由，覆盖当前任意状态。

**响应**（HTTP 202）：
```json
{ "ok": true, "action": "enable", "message": "已下发『全部允许上网』指令" }
```

### 2.2 一键禁网（禁止全部上网）

```
POST /api/network/disable
GET  /api/network/disable
```

等价于对所有被控端点击「禁止上网」：删除默认路由切断互联网（局域网/打印机保留）。

**响应**（HTTP 202）：
```json
{ "ok": true, "action": "disable", "message": "已下发『禁止上网』指令" }
```

### 2.2b 按 IP 开 / 禁单台

```
POST/GET /api/network/enable_ip?ip=192.168.1.21    允许该台上网
POST/GET /api/network/disable_ip?ip=192.168.1.21   禁止该台上网
```

只对指定 IP 的那台被控端生效，等价于在界面上只勾选该机再点「允许上网 / 禁止上网」。
常用于：复习平台某学生完成作答后**单独给他开网**，或单独禁某台。

- 必带 `ip` 查询参数，缺失返回 `400`。
- 与按钮一样会**记录该机的权威状态**（normal / disconnect）：即便该机当前离线，状态也已保存，等它下次上线注册时会自动应用。

**响应**（HTTP 202）：
```json
{ "ok": true, "action": "disable_ip", "ip": "192.168.1.21", "message": "已下发『禁止上网』指令: 192.168.1.21" }
```

调用示例：
```bash
curl "http://127.0.0.1:8770/api/network/disable_ip?ip=192.168.1.21"
curl "http://127.0.0.1:8770/api/network/enable_ip?ip=192.168.1.21&token=你的令牌"
```

### 2.3 查询状态（无需令牌）

```
GET /api/status
```

**响应**（HTTP 200）：返回所有被控端列表（在线在前），字段同界面。
```json
{
  "ok": true,
  "agents": [
    { "ip": "192.168.1.21", "hostname": "C4-03", "net_state": "normal",
      "is_online": true, "filter_active": false, "...": "..." }
  ]
}
```
`net_state` 取值：`normal`（正常）/ `whitelist` / `blacklist` / `disconnect`（已断网）。

### 2.4 健康检查（无需令牌）

```
GET /api/health
```
```json
{ "ok": true, "service": "network_control" }
```

> 兼容短路径：`/enable`、`/disable`、`/status`、`/health` 与上面的 `/api/...` 等价。

---

## 3. 响应约定

- 所有响应为 JSON，含 `ok` 字段（`true`/`false`）。
- 状态码：`202` 指令已接受、`200` 查询成功、`401` 令牌错误、`404` 路径不存在。
- **注意：接口为"触发即返回"**。返回 202 只表示指令已下发，被控端真正切换
  需要 1～2 秒。如需确认结果，稍后调用 `/api/status` 查看各机 `net_state`。
- 仅对**当前在线**的被控端生效；离线机重新上线后会自动恢复主控端记录的最近状态。

---

## 4. 鉴权（可选）

默认不校验令牌。设置令牌后，**写操作**（enable/disable）必须携带，三种方式任选其一：

```
Authorization: Bearer <token>
X-API-Token: <token>
?token=<token>          （拼在 URL 查询串里）
```

令牌错误返回 `401`。`/api/status` 与 `/api/health` 始终无需令牌。

---

## 5. 调用示例

### curl

```bash
# 一键禁网
curl -X POST http://127.0.0.1:8770/api/network/disable

# 一键开网
curl -X POST http://127.0.0.1:8770/api/network/enable

# 带令牌
curl -X POST http://127.0.0.1:8770/api/network/disable -H "Authorization: Bearer 你的令牌"

# 查询状态
curl http://127.0.0.1:8770/api/status
```

### PowerShell

```powershell
Invoke-RestMethod -Method Post -Uri http://127.0.0.1:8770/api/network/disable
Invoke-RestMethod -Method Post -Uri http://127.0.0.1:8770/api/network/enable
# 带令牌
Invoke-RestMethod -Method Post -Uri http://127.0.0.1:8770/api/network/disable `
  -Headers @{ Authorization = "Bearer 你的令牌" }
```

### Python

```python
import urllib.request

def call(action, token=""):
    url = f"http://127.0.0.1:8770/api/network/{action}"
    req = urllib.request.Request(url, method="POST")
    if token:
        req.add_header("Authorization", f"Bearer {token}")
    with urllib.request.urlopen(req, timeout=5) as r:
        print(r.read().decode())

call("disable")   # 禁网
call("enable")    # 开网
```

### 浏览器 / 任意语言

直接 GET 访问即可（无令牌时）：
`http://127.0.0.1:8770/api/network/disable`

## 6. 配置修改

接口参数存放在主控端数据库 `controller.db` 的 `settings` 表，键名：

| 键名 | 默认 | 说明 |
|------|------|------|
| `api_host` | `127.0.0.1` | 监听地址；改 `0.0.0.0` 允许局域网其他机器调用 |
| `api_port` | `8770` | 监听端口 |
| `api_token` | 空 | 访问令牌；非空即开启鉴权 |

在主控端「**设置**」标签页直接填写「控制接口地址 / 端口 / 令牌」三项，点「保存设置」即可。
修改后**重启主控端**生效。

> 安全提示：若把 `api_host` 改成 `0.0.0.0` 开放到局域网，**务必同时设置 `api_token`**，
> 否则同网段任何人都能开关网络。

## 7. 实现位置（开发参考）

- `controller/server/api_server.py` — `ControlAPIServer`，标准库 `http.server`，独立 daemon 线程。
- `controller/gui/main_window.py`：
  - `_start_api_server()` 读取设置并启动接口；
  - 回调通过 `pyqtSignal`（`api_enable` / `api_disable` / `api_enable_ip` / `api_disable_ip`）排队到 Qt 主线程；
  - `_on_api_enable()` → `_on_allow_all([])`，`_on_api_disable()` → `_on_disconnect([])`；
  - `_on_api_enable_ip(ip)` → `_on_allow_all([ip])`，`_on_api_disable_ip(ip)` → `_on_disconnect([ip])`（复用同一套处理，含状态保存）；
  - `closeEvent()` 中 `stop()` 关闭接口。

跨线程模型与项目既有约定一致：HTTP 线程**只 emit 信号**，真正的 WebSocket 下发和
数据库写入都在 Qt 主线程完成，避免 sqlite 跨线程与并发问题。
