# 主控端 默认配置说明

主控端的配置**不是文件**，而是保存在程序同目录自动生成的 `controller.db`（SQLite 的 `settings` 表）里。
**首次运行 `主控端.exe` 会自动创建 `controller.db` 并写入下面的默认值**，之后在程序「设置」标签页里修改即可（改完重启主控端生效）。

## 默认值

| 配置项 | 默认值 | 说明 |
|---|---|---|
| `controller_port` | `8765` | WebSocket 监听端口。**必须**和被控端 `config.json` 里 `controller_url` 的端口一致 |
| `upstream_dns` | `114.114.114.114` | 上游 DNS，断网/白名单时给被控端解析用 |
| `lan_subnets` | `192.168.1.0/24` | 教室局域网网段（断网时仍放行，保证能连到主控端）。用「网段计算器.html」按本机 IP+掩码算 |
| `tray_password_hash` | `admin123` 的 SHA256 | 被控端托盘退出密码（可远程下发给被控端） |
| `unlock_password` | `admin123` 的 SHA256 | 拔网线锁屏的解锁密码 |
| `api_host` | `127.0.0.1` | HTTP 控制接口监听地址（详见「接口说明.md」） |
| `api_port` | `8770` | HTTP 控制接口端口 |
| `api_token` | 空 | HTTP 接口令牌；开放到 `0.0.0.0` 时**务必**设置 |

> `admin123` 对应的 SHA256 = `240be518fabd2724ddb6f04eeb1da5967448d7e831c08c8fa822809f74c720a9`

## 首次部署要改的两项

1. **`lan_subnets`**：改成你教室实际网段（和被控端保持一致）。
2. **密码**：如需更安全，改掉默认 `admin123`。

## 注意

- `controller.db` 删除后会用上述默认值重新生成（白/黑名单、历史状态也会一并清空）。
- 端口 `8765` 改了的话，**所有被控端的 `config.json` 也要同步改**，否则连不上。
