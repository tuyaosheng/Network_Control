# 局域网网络控制系统 — 使用说明

适用版本：2.2　　适用系统：Windows 10 / 11

> 2.2 新增：开机默认断网（fail-closed）、断网仍可被主控端管理、按 IP 单台开/禁网接口、拔网线锁屏修复。

---

## 一、系统概述

本系统用于电脑室局域网网络访问管理。教师机运行**主控端**，通过图形界面统一管理所有学生机的上网行为；学生机安装**被控端**服务，开机自动启动，无需任何操作。

支持的控制模式：

| 模式 | 效果 |
|---|---|
| 白名单过滤 | 学生只能访问白名单内的指定网站，其余网站全部屏蔽 |
| 黑名单过滤 | 屏蔽黑名单内的网站，其余网站照常访问 |
| 一键断网 | 学生断开互联网，局域网内通信（打印机、内网系统等）不受影响 |
| 全部允许上网 | 停止所有过滤，恢复正常访问 |

> **防绕过说明**：白名单模式通过修改路由表实现，仅为白名单域名解析出的 IP 添加主机路由（/32），其余出站流量无路由可达，自然阻断。学生即使手动修改网卡 DNS 或直接输入 IP 地址也无法访问非白名单站点（未经允许的 IP 没有路由）。黑名单模式仅用 DNS 拦截，不修改路由表。

## 二、部署文件说明

打包后的部署文件位于 `dist\` 目录：

```
dist\
  主控端.exe      → 复制到教师机，双击运行
  被控端.exe      → 复制到每台学生机
  config.json    → 被控端配置文件，须与被控端.exe 放在同一目录
```

> `controller.db`（主控端数据库）会在主控端首次运行时自动生成，白名单/黑名单规则和设置均保存于此，无需手动创建。

## 三、初次部署

### 3.1 教师机（主控端）

1. 将 `主控端.exe` 复制到教师机任意目录（建议 `C:\NetControl\`）
2. 双击运行，首次运行会自动创建数据库
3. 确认底部状态栏显示"服务器运行中 | 端口 8765"

### 3.2 学生机（被控端）

**第一步：修改配置文件**

用记事本打开 `config.json`，修改以下字段：

```json
{
  "controller_url": "ws://192.168.1.100:8765",
  "upstream_dns":   "114.114.114.114",
  "tray_visible":   true,
  "lan_subnets":    ["192.168.1.0/24"],
  "tray_password_hash":   "240be518fabd2724ddb6f04eeb1da5967448d7e831c08c8fa822809f74c720a9",
  "unlock_password_hash": "240be518fabd2724ddb6f04eeb1da5967448d7e831c08c8fa822809f74c720a9"
}
```

| 字段 | 说明 |
|---|---|
| `controller_url` | **必改，且必须是 IP**（不能用域名）。将 IP 换成教师机的局域网 IP，端口默认 8765。开机默认断网时靠 IP 才能保留到主控端的路由并连上 |
| `upstream_dns` | 上游 DNS，填学校/运营商的 DNS 服务器，默认 114.114.114.114 |
| `tray_visible` | 是否显示右下角托盘图标，`false` 可对学生完全隐藏 |
| `lan_subnets` | 局域网网段，此网段流量不受断网/过滤影响，须与实际一致 |
| `tray_password_hash` | 托盘图标"退出"密码的 SHA256 值，默认密码 `admin123` |
| `unlock_password_hash` | 拔网线锁屏的解锁密码 SHA256 值，默认密码 `admin123` |

> 两个密码的 hash 值也可以直接在主控端「设置」页修改，下次推送规则时自动同步到被控端，无需手动编辑 config.json。

**第二步：安装服务（需管理员权限）**

将 `被控端.exe` 和 `config.json` 复制到学生机同一目录（建议 `C:\NetControl\`），然后在该目录以管理员权限打开命令提示符：

```
被控端.exe install
被控端.exe start
```

安装完成后服务即刻运行，之后开机自动启动。

**服务管理命令**（管理员权限）：

```
sc.exe start NetControlAgent    # 启动
sc.exe stop  NetControlAgent    # 停止
被控端.exe remove               # 卸载
```

**控制按钮说明：**

**被控端列表列说明：**

| 列 | 说明 |
|---|---|
| □（表头全选） | 点击表头 checkbox 可全选/取消全选所有在线机器 |

| 主机名 | 计算机名称（由被控端注册时上报） |

**列排序：** 点击任意列标题可按该列排序（再次点击切换升序/降序），▲/▼ 指示当前排序列和方向。IP 地址按数值大小排序，网络状态按 正常→白名单→黑名单→已断网→已断开 顺序排序。

**已断开机器：** 曾连接过但已与主控端断开的被控端会保留在列表中，行显示为灰色，状态列显示"● 已断开"，checkbox 禁用无法勾选。重新上线后自动恢复正常显示并保留历史浏览记录。

## 五、日常使用流程

### 5.1 配置白名单

1. 点击右侧「规则管理」→「白名单」标签
2. 点击「+ 添加条目」，输入名称（如：百度）和域名（如：`baidu.com`）
3. 添加后左侧「启用」列默认打勾；取消勾选则该条目临时禁用，不会下发
4. 支持通配符：`*.baidu.com` 表示 baidu.com 及其所有子域名
5. 双击「名称」或「域名」单元格可直接修改，回车或点击其他位置保存

> **网站显示不全怎么办？** 现代网站的图片、样式、字体等资源往往来自其他域名（CDN）。按 `F12` 打开浏览器开发者工具 → Network 标签，查看失败的请求是哪些域名，逐一加入白名单。

### 5.2 启动白名单过滤

点击「▶ 启动白名单」按钮。操作会同时下发规则并启用过滤，无需分两步操作。此时学生只能访问白名单内已启用的域名对应的 IP。

**勾选逻辑**：
- 未勾选任何机器或勾选了全部 → 所有在线被控端同时生效
- 只勾选了部分机器 → 仅对勾选的机器生效

**实现原理**：被控端收到指令后，先临时恢复默认路由以供 DNS 解析，解析出所有白名单域名的 IP 后立即批量添加主机路由（/32），最后删除默认路由。整个切换过程约 2～5 秒；期间有短暂的完全联网窗口（用于域名解析），之后即进入严格白名单模式。

### 5.3 启动黑名单过滤

与白名单流程相同，在「规则管理」→「黑名单」中配置要屏蔽的域名，然后点击「▶ 启动黑名单」。

黑名单模式**仅通过 DNS 拦截**，不修改路由表，适合屏蔽特定网站但不限制其他访问的场景。

### 5.4 一键断网 / 恢复

- 顶部红色按钮「⛔ 禁止上网」：删除默认路由，所选学生机立即断开互联网
- 绿色按钮「✅ 全部允许上网」：恢复默认路由，停止所有过滤规则

断网后局域网内通信（打印机、内网教学平台、文件共享）不受任何影响。

### 5.5 查看浏览记录

- 「最近访问」列显示该机器最后一次访问的域名
- 右键某台机器 → 「查看浏览记录」：弹出详细历史（最近 30 条，含时间戳）
- 记录基于 DNS 查询，仅当被控端处于过滤模式（白名单或黑名单）时才会采集

### 5.6 多机批量操作

1. 点击表头 checkbox（□）全选所有在线机器，或手动勾选部分机器
2. 对勾选的机器统一执行断网、恢复、启动过滤等操作
3. 再次点击表头 checkbox 可取消全选
4. 离线机器（灰色行）不可勾选，不受批量操作影响

## 六、设置说明

点击右侧「设置」标签：

| 设置项 | 说明 |
|---|---|
| 主控端端口 | WebSocket 监听端口，默认 8765。**修改后需重启主控端**，学生机 config.json 的 controller_url 也须同步修改 |
| 上游 DNS | 被控端在过滤时转发 DNS 查询的服务器，填学校或运营商 DNS；白名单模式下也用于解析白名单域名 |
| 局域网段 | 不受过滤/断网影响的本地网段，多个用英文逗号分隔，如 `192.168.1.0/24,192.168.2.0/24` |
| 被控端托盘退出密码 | 学生右键托盘图标退出时需输入此密码，留空则不修改 |
| 被控端锁屏解锁密码 | 拔网线后锁屏界面的解锁密码，留空则不修改 |

> 修改密码后点「保存设置」，下次推送规则（启动白名单/黑名单）时新密码会自动同步到所有被推送的学生机，无需手动处理 config.json。

## 七、拔网线锁屏功能

当被控端检测到网线被拔出（所有物理/无线网卡均断开）时：

1. 立即弹出**全屏锁定界面**（深蓝背景，无法最小化或关闭）
2. 显示 60 秒倒计时
3. 三种解锁方式：
   - **插回网线**：检测到网络恢复后自动解锁
   - **输入密码**：在文本框输入锁屏解锁密码后点确认
   - **超时不操作**：60 秒内未解锁则**自动关机**

> 锁屏界面屏蔽了 Esc 和 Alt+F4 按键，学生无法通过常规方式关闭。

## 九、开机默认断网（fail-closed，重要）

为杜绝学生机「开机到被纳管之间」的自由上网空当，被控端采用**故障安全**策略：

1. **服务随开机立即自启**（安装时已配置为 `auto`，不再延迟 1~2 分钟）；
2. 被控端启动后**先默认断网**（删默认路由），但**保留到主控端的路由**，因此仍能连上主控端；
3. 连上后，主控端在该机注册时**下发它的权威状态**（= 老师上次的设定，保存在 `controller.db`）：
   - 上次是「正常 / 从没设过」→ 主控端自动发「允许上网」→ **恢复上网**；
   - 上次是「断网 / 白名单 / 黑名单」→ 维持对应状态。

**注意事项：**
- **主控端要先开着**，学生机开机才能连上来拿到「允许上网」；主控端没开 → 学生机一直断网（这正是 fail-closed 的目的）。应急时可在学生机托盘输密码退出被控端以恢复网络。
- `config.json` 的 `controller_url` **必须是 IP**（断网状态下无法解析域名）。
- 权威状态以**老师的操作**为准；被控端开机那一下的临时断网**不会**被当成老师的设定，所以「上次是正常」的机器开机后会自动开网，不用手动点。

## 十、常见问题

**Q：学生机没有出现在列表里？**
检查：① 被控端服务是否运行（任务管理器 → 服务 → NetControlAgent）；② config.json 中的 IP 是否正确；③ 教师机 Windows 防火墙是否放行了 8765 端口（入站规则）。

**Q：列表里的机器显示灰色"已断开"是什么意思？**
被控端曾成功连接过主控端，但当前与主控端的 WebSocket 连接已断开（可能是被控端服务未运行、网络故障或学生机关机）。这类记录会永久保留直到主控端重启，被控端重新上线后自动恢复正常状态。

**Q：白名单的网站打开但图片/样式缺失？**
该网站的静态资源托管在其他 CDN 域名上。用 F12 开发者工具查看失败请求的域名，加入白名单即可。

**Q：启动白名单时有短暂的完全联网现象，正常吗？**
正常。白名单切换需要先临时恢复网络以解析域名 IP，解析完成后立即批量添加主机路由并断开默认路由。整个过程约 2～5 秒，之后即进入严格白名单模式。

**Q：学生修改网卡 DNS 能绕过过滤吗？**
白名单模式下不能绕过。白名单基于路由表（主机路由 /32）实现，非白名单 IP 没有出站路由，学生改 DNS 也无法访问未在路由表中的地址。黑名单模式仅靠 DNS 拦截，修改 DNS 可以绕过，建议高安全场景使用白名单模式。

**Q：学生直接输 IP 地址访问能绕过吗？**
白名单模式下不能。路由表只为白名单域名解析出的 IP 开放了主机路由，直接输入其他 IP 的数据包同样没有路由可达，会被丢弃。

**Q：主控端关掉后学生机还在过滤吗？**
会。路由表和 DNS 服务都在学生机本地运行，主控端断开不影响已启用的过滤。学生机断线重连后会自动重新注册，主控端重新上线时也能看到历史离线记录。

**Q：学生机开机后，主控端不点也不开网、一直断网，正常吗？**
分两种情况：① **主控端没开**——属正常（fail-closed 设计），学生机连不上主控端就保持断网，开主控端后会自动放行；② **主控端开着却仍不开网**——请确认 `controller_url` 是该教师机的 IP、两端同网段/端口、主控端能看到该机上线。只要老师上次没把它设成「禁止上网」，连上后会自动开网，无需手动点。

**Q：把某台设为「禁止上网」后，它重启还会是断网吗？**
是。主控端记录了该机的权威状态（断网），它重启连上来后会自动恢复断网。要让它上网，对它点「允许上网」（或调用 `enable_ip` 接口）即可，这会把权威状态改回「正常」。

**Q：学生强制结束被控端进程会怎样？**
服务配置了失败自动重启策略（60 秒内重启，最多 3 次）。白名单路由规则在重启期间仍然有效，网络状态不受影响。

**Q：如何确认某台学生机的过滤已生效？**
在主控端列表查看该机器的「网络状态」列；也可以在学生机尝试访问一个不在白名单内的网站，应显示"无法访问此网站"。
