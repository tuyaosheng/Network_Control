# Network Control · 局域网网络访问控制系统

教师机（**主控端**）通过 WebSocket 统一管理学生机（**被控端**）的网络状态，适用于电脑室 / 机房等局域网场景。一键在 **正常上网 / 白名单过滤 / 黑名单过滤 / 断网** 四种状态间切换，并可结合复习平台「完成作答后自动开网」。

> 本仓库为**发布包**：包含两端打包好的 exe、默认配置、安装脚本、使用与接口说明，以及完整源码。拿来即用，也可自行改源码重新打包。

---

## ✨ 功能特性

- **四种网络状态**：正常 / 白名单（只放行指定网站）/ 黑名单（只封禁指定网站）/ 断网。
- **批量 / 单台控制**：主控端勾选多台学生机一键切换；也支持按单台 IP 精确开/禁网。
- **开机默认断网（fail-closed）**：学生机开机即先断网，连上主控端、按上次状态自动放开/保持，杜绝开机空当的自由上网。
- **断网仍可管理**：断网时保留到主控端的路由，主控端依旧能看到该机并随时放行（即使跨网段）。
- **白名单双层过滤**：DNS 拦截 + 路由表动态放行，绕开「Windows 封锁规则优先级高于放行」的坑。
- **断线自动重连 + 状态恢复**：学生机（含还原卡重启）回到干净状态后，主控端按记录的权威状态自动恢复。
- **拔网线锁屏**：检测到网线被拔出时弹出全屏锁屏，需密码解锁；插回网线自动消失，超时未解锁则关机。
- **浏览监控**：主控端可查看各学生机最近访问的域名。
- **HTTP 控制接口**：供其它程序一键开网 / 禁网、或按 IP 单台开/禁（如复习平台「完成作答后自动开网」）。
- **系统托盘**：学生机托盘图标，退出需 SHA256 密码（应急后门，可恢复网络）。

## 🚀 快速开始

### 一、主控端（教师机）

1. 把 **主控端（教师机）** 文件夹拷到教师机任意位置。
2. 双击 `启动主控端.bat`（或直接 `主控端.exe`），普通权限即可，**无需安装**。
3. 首次运行自动生成 `controller.db`。打开「设置」标签页确认：
   - **端口** 默认 `8765`（要和被控端 `config.json` 一致）；
   - **局域网网段 `lan_subnets`**：用同目录 `网段计算器.html` 按教师机 IP + 子网掩码算出（如 `192.168.1.0/24`）。
4. 在「白/黑名单」里维护域名规则。

### 二、被控端（学生机）

1. 编辑 **被控端（学生机）/config.json**，把 `controller_url` 的 IP 改成**教师机 IP**：
   ```json
   { "controller_url": "ws://192.168.1.100:8765", ... }
   ```
2. 把 `被控端.exe`、`config.json`、`install_agent.bat` **放进学生机本地同一文件夹**（先解压，别在压缩包里直接跑）。
3. **管理员**身份运行 `install_agent.bat`（脚本会自动申请 UAC）。它会：装服务 → 启动 → 写 `install_agent.log`。
4. 验证：学生机执行 `sc query NetControlAgent`，看到 `STATE: 4 RUNNING`；再到主控端面板看到这台机器上线即成功。

> 卸载：运行 `uninstall_agent.bat`，或管理员 `被控端.exe remove`。

## ⚙️ 配置说明

| 端 | 配置位置 | 关键项 |
|---|---|---|
| 主控端 | `controller.db`（首次运行自动生成，在 GUI「设置」改） | `controller_port`(8765) · `upstream_dns` · `lan_subnets` · 密码 · `api_host/port/token` |
| 被控端 | `config.json`（exe 同目录） | `controller_url`（教师机地址，**必改**）· `upstream_dns` · `lan_subnets` · `tray_password_hash` |

- 两端的 **端口** 和 **网段** 必须一致。
- 默认密码 `admin123`（SHA256: `240be518…c720a9`），生产环境建议修改。

## 🌐 工作模式（被控端网络状态）

| 状态 | DNS | 防火墙 | 路由 |
|------|-----|--------|------|
| `normal` | DHCP DNS | 全部放行 | 默认网关存在 |
| `whitelist` | 127.0.0.1 | 默认封锁 + 仅放行白名单 | 删默认路由，只留主机路由 |
| `blacklist` | 127.0.0.1 | 清空（仅 DNS 拦截） | 默认网关存在 |
| `disconnect` | 上游 DNS | 清空 | 删默认路由 |

## 🔐 开机行为（fail-closed，默认断网）

为杜绝学生机开机到被纳管之间的「自由上网空当」，被控端采用**故障安全**策略：

1. **服务立即自启**（`auto`，跟随开机，不再延迟 1~2 分钟）；
2. 被控端启动后**先默认断网**（删默认路由），但**保留到主控端的路由**，因此仍能连上主控端；
3. 连上后主控端在其注册时**下发该机的权威状态**（老师上次设定，存于 `controller.db`）：
   - 上次是「正常 / 没设过」→ 主控端发「放行」→ **自动恢复上网**；
   - 上次是「断网 / 白名单 / 黑名单」→ 维持对应状态。

要点：
- **主控端需开着**，学生机才能连上来拿到「放行」；主控端没开 → 学生机保持断网（这正是 fail-closed 的目的）。应急可在学生机托盘输密码退出被控端以恢复网络。
- 被控端 `config.json` 的 `controller_url` **必须是 IP**（不能用域名）——断网状态下无法做 DNS 解析，靠 IP 才能保留到主控端的路由并连上。
- 权威状态以**老师的操作**为准（按钮 / API 设置时写入），被控端开机那一下的临时断网**不会**被当成老师的设定。

## 🔌 HTTP 控制接口（主控端）

主控端启动时附带一个 HTTP 接口，供其它程序一键控制：

```
POST/GET /api/network/enable          一键开网（全部被控端）
POST/GET /api/network/disable         一键禁网（全部被控端）
POST/GET /api/network/enable_ip?ip=   按 IP 开单台
POST/GET /api/network/disable_ip?ip=  按 IP 禁单台
GET      /api/status                  被控端列表及状态
GET      /api/health                  健康检查
```

默认监听 `127.0.0.1:8770`，无令牌。开放到 `0.0.0.0` 时务必设置 `api_token`。完整说明见 `主控端（教师机）/接口说明.md`。

**💡 典型用法：配合其他教学软件「学生完成任务即开网」**

把本接口和你的教学/作业软件联动，即可实现「没做完不给上网，做完自动开网」：

```bash
# 1) 上课先全部禁网，让学生专注做任务
curl "http://127.0.0.1:8770/api/network/disable"

# 2) 你的软件判定某学生完成时，按他机器的 IP 单独开网
curl "http://127.0.0.1:8770/api/network/enable_ip?ip=192.168.1.21"
```

完成的人自动上网、没完成的继续断网，老师无需手点。任何能发 HTTP 请求、能拿到学生机 IP 的软件都可集成。

## 🛠 从源码构建

进入对应的 `源码/` 目录：

```bat
:: 安装依赖（两端通用）
pip install -r requirements.txt

:: 主控端
build_controller.bat        :: → dist\主控端.exe

:: 被控端
build_agent.bat             :: → dist\被控端.exe + config.json
```

调试运行（不打包）：`run_controller.bat` / `run_agent_debug.bat`（被控端需管理员）。

**技术栈**：Python · PyQt6（主控端 GUI / 锁屏）· websockets(<13) · dnslib（DNS）· pywin32（Windows 服务）· PyInstaller（打包）。

> ⚠️ `websockets` 必须 `<13.0`：13+ 删除了 `WebSocketServerProtocol`，服务端直接依赖该类型。

## ❓ 常见问题

- **服务列表里找不到？** 用 `sc query NetControlAgent` 确认；服务显示名是中文「网络控制-被控端」。多半是安装时 UAC 没点「是」，导致没真正安装——用管理员终端跑 `被控端.exe install` 看报错。
- **学生机连不上主控端？** 检查 `config.json` 的 `controller_url`（IP/端口）、教师机防火墙是否放行 8765、两端是否同网段。
- **白名单放行了但仍打不开网站？** 该网站可能用了未在白名单中的第三方域名（CDN/接口域名），需一并加入。
- **bat 双击乱码？** 本仓库 bat 已用 GBK 编码，适配中文 Windows；若自己改动请保持 GBK 保存。

## 🔒 安全提示

- 默认密码、空 API 令牌仅供测试，正式部署请修改。
- HTTP 接口默认仅本机可访问（127.0.0.1）；如需跨机调用并开放到 `0.0.0.0`，**必须**设置 `api_token`。
- 本系统用于**授权的局域网教学管理**场景，请勿用于未授权的网络干扰。
