# 被控端 接口说明（通信协议）

被控端通过 **WebSocket** 连接主控端（地址在 `config.json` 的 `controller_url`），双方收发 **UTF-8 的 JSON 消息**。被控端本身**不对外开放任何端口/接口**，只主动连主控端；唯一的本地监听是 `127.0.0.1:53` 的内置 DNS（仅本机过滤用，不对外）。

> 如果你要的是「用其它程序一键开网/禁网」的 HTTP 接口，那个在**主控端**侧，见主控端文件夹里的「接口说明.md」。

## 1. 消息类型

```
被控端 → 主控端：
  register        连接后立即发送，携带 主机名 / IP / MAC
  heartbeat       心跳（每 20 秒），携带过滤状态和网络状态
  status          状态上报（响应主控端 get_status）
  ack             操作确认
  browsing_update 浏览域名上报（每 15 秒，最近 DNS 查询列表）

主控端 → 被控端：
  update_rules    下发规则（域名列表、模式、LAN 网段、DNS、密码 hash）
  set_filter      启用/禁用过滤（模式 whitelist / blacklist）
  disconnect      断网指令
  reconnect       恢复上网指令
  get_status      请求状态上报
```

## 2. 关键消息字段

**update_rules（主控端下发规则）**
```json
{
  "type": "update_rules",
  "domains":          ["baidu.com", "*.baidu.com"],
  "lan_subnets":      ["192.168.1.0/24"],
  "controller_ip":    "192.168.1.100",
  "upstream_dns":     "114.114.114.114",
  "mode":             "whitelist",
  "tray_pwd_hash":    "<sha256>",
  "unlock_pwd_hash":  "<sha256>"
}
```

**set_filter**
```json
{ "type": "set_filter", "enabled": true, "mode": "whitelist" }
```

**browsing_update（被控端上报最近浏览）**
```json
{
  "type": "browsing_update",
  "domains": [
    { "domain": "www.baidu.com", "ts": "10:23:45" }
  ]
}
```

## 3. 网络状态值

| 状态 | DNS | 防火墙 | 路由 | 含义 |
|------|-----|--------|------|------|
| `normal` | DHCP DNS | 全部放行 | 默认网关存在 | 正常上网 |
| `whitelist` | 127.0.0.1 | 默认封锁 + 仅放行白名单（NC_ 规则） | 删默认路由，只留主机路由 | 白名单过滤中 |
| `blacklist` | 127.0.0.1 | 清空（仅靠 DNS 拦截） | 默认网关存在 | 黑名单过滤中 |
| `disconnect` | 上游 DNS | 清空 | 删默认路由 | 已断网 |
| `offline` | — | — | — | 仅主控端侧：与被控端连接已断开 |

## 4. 行为要点

- **断线自动重连**：被控端与主控端断开后会持续重连（`ws_client.py`）。
- **重连状态恢复**：被控端（如还原卡重启）重新 `register` 时，主控端若记录其上次非 `normal`，会自动重新下发规则，恢复过滤/断网状态。
- **协议常量定义**：源码 `源码/shared/protocol.py`。

（更详细的双层过滤、状态机实现见 `源码/技术手册.md`。）
